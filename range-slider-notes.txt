Handle picking (how knobs are grabbed)
File: src/ui/components/RangeSlider.tsx
- The knob itself is the hit target (no track click-to-move).
- Drag starts only on knob mouse/touch down.
- While dragging, document-level listeners handle pointer move and release.

Code (core flow)
const startDrag = (knob, event) => {
  event.preventDefault();
  setDraggingKnob(knob);
};

useEffect(() => {
  if (draggingKnob) {
    document.addEventListener('mousemove', handleDragMove);
    document.addEventListener('touchmove', handleDragMove, { passive: false });
    document.addEventListener('mouseup', endDrag);
    document.addEventListener('touchend', endDrag);
  }
  return () => { ...remove listeners... };
}, [draggingKnob, ...]);

handleDragMove:
- reads track bounding box (trackRef.getBoundingClientRect())
- converts pointer x to percentage, clamps 0..100
- converts percentage to value and clamps to min/max
- updates left/right values based on which knob is active

Tick markers (how dots are rendered)
File: src/ui/components/RangeSlider.tsx
- Each tick is generated from dotValues.
- Each tick is placed by converting value -> percentage of the track.

{dotValues.map((dotValue, index) => {
  const dotPercentage = valueToPercentage(dotValue);
  return (
    <div
      key={`dot-${index}`}
      className="slider-track-dot"
      style={{ left: `${dotPercentage}%` }}
    />
  );
})}
